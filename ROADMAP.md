# 实验路线图

## 目标

对量潮科技数十个飞书知识库进行语义分析，从内容层面理解不同时期的知识重心、主题演变与信息分布，为知识治理提供语义维度的洞察。

## 关键问题

- 不同时期的知识库在聊什么话题？主题如何演变？
- 哪些领域知识密集、哪些领域存在空白？
- 文档之间是否存在语义重叠或重复？
- 不同团队的知识领域是否有交叉或孤岛？
- 如何从语义层面指导知识库的整合与治理？

## 当前阶段：语义摸底（v0.1.x）

对全量知识库文档进行语义层面的扫描与聚类，建立主题地图。

### 待办

- [ ] **文档内容采集** — 批量拉取全量知识库的文档内容（正文 + 标题）
- [ ] **语义向量化** — 对文档进行 embedding，建立向量索引
- [ ] **主题聚类** — 基于向量相似度进行聚类，识别各时期的核心主题簇
- [ ] **主题演化分析** — 按时间维度排列主题，观察知识重心的迁移轨迹
- [ ] **语义报告** — 输出主题地图可视化 + 演化趋势图 + 空白领域标注

### 检验标准

- 主题地图能清晰展示 5～10 个核心主题簇及其关联
- 演化趋势能回答"不同时期文档在关注什么"
- 报告标注出明显的信息孤岛或空白领域

## 后续方向

### 方向 1 — 语义治理建议

- 基于主题分布提出知识库合并、拆分、归档的语义依据
- 识别高价值但分散的知识领域，建议集中化管理

### 方向 2 — 智能检索与导航

- 基于语义索引构建知识库级联搜索
- 按主题而非目录结构组织知识导航

### 方向 3 — 持续语义监控

- 新文档入库后自动归入主题簇，发现新主题及时告警
- 跟踪治理动作后主题分布的变化趋势
